Notation & definitions¶

Molecular phylogenetics requires being somewhat comfortable with probability theory and statistics. For clarity's sake, let's fix/refresh some notation and definitions:

An event $A$ can refer to any proposition, e.g. "Two birds are born in the next five minutes". Typically we associate events with a sample space, which could here be all possible numbers of birds.

In many cases, an event $A$ can be associated with a probability $P(A)$ (which you intuitively grasp, but can be defined in different ways, leading to philosophical minefields). By $P(A|B)$ we denote the conditional probability, that is the probability of event $A$ given that some other event $B$ occurred.

A random variable $X$ is a mapping from events to a real number. With a random variable, we asociated a function $p(X)$, known as the probability mass/density function (pmf/pdf) of the random variable $X$.

The data $D$ will always be a set of aligned molecular sequences, either amino acids, nucleotides or codons. An alignment represents a hypothesis of homology, i.e. it states our beliefs on which characters in a bunch of sequences have been derived from a common ancestral character.

Maximum Likelihood¶

Statistical principles¶

The likelihood is a function of the parameters of interest $\theta$ and is related to the probability of observing data $D$ under some probabilistic model $\mathcal{M}$. We write:

$$\mathcal{L}(\theta|D,\mathcal{M}) = P(D|\theta,\mathcal{M})$$

Maximum likelihood estimation is a technique for estimating parameters (i.e. coming up with an estimate $\hat{\theta}$ for the unknown parameter(s) $\theta$) of a probabilistic model where we try to find the values of $\theta$ that correspond to a maximal likelihood, that is

$$\hat{\theta} = \mathrm{argmax}_{\theta}\mathcal{L}(\theta|D,\mathcal{M}) $$

Example: For the last 10 years you observed whether on the first of May there was a magpie in your garden, you wrote 1 if there was at least one and 0 if there was none. Your data set looks as follows:

In [3]:
magpies = [0,1,1,1,0,1,1,0,0,1]

What is the maximum likelihood estimate for the probability of having a magpie in your garden on the first of May?

Our random variable $X$ takes on two values, either $X = 1$ or $X = 0$. The probabilistic model $\mathcal{M}$ we use is a very simple one: every year on the first of May there is either a magpie in your garden ($X = 1$), which happens with probability $P(X = 1|\mathcal{M}) = \theta$, or there is no magpie ($X = 0$), with probability $P(X = 0| \mathcal{M}) = 1-\theta$ (in technical terms, your model is a so-called Bernouilli process).

Recall that the likelihood is

$$\mathcal{L}(\theta|D,\mathcal{M}) = P(D|\theta,\mathcal{M})$$

Since we assume that the different observations are independent, $P(D|\theta, \mathcal{M})$ is simply

\begin{eqnarray} \mathcal{L}(\theta|D,\mathcal{M}) &=& P(D|\theta, \mathcal{M}) \\ &=& p(X_1 = 0, X_2 = 1, \cdots| \theta, \mathcal{M}) \\ &=& p(X_1 = 0| \theta, \mathcal{M})p(X_2 = 1| \theta, \mathcal{M})\cdots \\ &=& \theta^6(1-\theta)^4 \end{eqnarray}

Since this is a function of $\theta$, we can make the following plot:

Phylogenetic likelihood¶

Markov models of nucleotide substitution and the rate matrix¶

In order to do statistical phylogenetics, we need a probabilistic model of sequence evolution. This is a model that allows us to associate a probability to the event that sequence $a$ evolved to sequence $b$ over a certain time $t$. These are the so-called substitution models that you have discussed (e.g. JC69, K80, F81, HKY85, GTR, ...).

Substitution models are Markov models that describe transitions between states (e.g. A, T, C, G in the case of nucleotide sequences) at a particular alignment site.

The substitution model is fully determined by it's rate matrix $Q$. The entries in $Q$ are simply the rates at which substitution occur from one nucleotide to another, which is a random process (a Poisson process) with waiting times between two substitution events having an exponential ditribution. This is very similar to for example radio-active decay.

$$Q = \begin{bmatrix} - & \lambda_{GA} & \lambda_{CA} & \lambda_{TA} \\ \lambda_{AG} & - & \lambda_{CG} & \lambda_{TG} \\ \lambda_{AC} & \lambda_{GC} & - & \lambda_{TC} \\ \lambda_{AT} & \lambda_{GT} & \lambda_{CT} & - \end{bmatrix}$$

def simulateQ(Q, seq="AAAAA", t=2):
    # just a map from nucleotides to indices of the matrix (and reverse)
    charmap = {"A":0, "G":1, "C":2, "T":3}         
    invmap  = {v: k for k, v in charmap.items()}
    newseq = ""                                    
    # loop over independently evolving sites
    for site in seq:
        # get indices of the nucleotide and the ones it can substitute to
        state = charmap[site]   
        other = list(set(range(4)) - set([state]))
        # start with a time `t` and draw the time until the first substitution
        t_  = t
        t_ -= np.random.exponential(sum(Q[other, state]))  
        while t_ > 0: 
            # pick a state to substitute to
            psubs = Q[other, state]/sum(Q[other, state])
            state = np.random.choice(other, p=psubs)  
            other = list(set(range(4)) - set([state])) 
            # draw the waiting time for the next 
            t_ -= np.random.exponential(sum(Q[other, state])) 
        # add character to the new sequence
        newseq += invmap[state]
    print("alignment:")
    print("\tseq(t={:5.1f}) ".format(0) + seq)
    print("\tseq(t={:5.1f}) ".format(t) + newseq)
    return newseq

Associated with this rate matrix is the transition probability matrix

$$ P(t) = e^{Qt} $$

You might not know how to compute a matrix exponential, but all you need to know is that it's not that hard. The transition probability matrix is very similar to $Q$, but does not give the rates of the exponential ditributions of the substitution process, but probabilities of transitions. That is, $P_{ij}(t)$ gives the probability that nucleotide $i$ is substituted by nucleotide $j$ over time $t$.

Sequence likelihood on a phylogenetic tree¶

The markov model of the substitution process and its associated transition probability matrix provides us the probabilistic model $\mathcal{M}$ needed to compute the likelihood of a phylogenetic tree given sequence data $D$. More precisely, the probabilistic model is the substitution process described above, but now running along the branches of a tree structure. At every split in the tree, the substitution process splits into two independent new subtitution processes. Our goal is now to, based on a set of sequences observed at the tips of the tree, compute the likelihood of the tree topology $T$, its branch lengths $b$, and the parameters of the substitution process $\theta$, that is

$$\mathcal{L}(T, b, \theta|D, \mathcal{M}) = P(D|T, b, \theta, \mathcal{M})$$

It turns out that te only thing we need to do this is the transition probability matrix $P = e^{Qt}$ described above, and a clever algorithm (Felsenstein's pruning algorithm).

Example: Here is a very simple (and not the most efficient implementation) of Felsenstein's pruning algorithm. It is implemented in a recursive fashion. Try to understand why it is able to compute the likelihood for a given site given the tree, branch lengths and substitution model.

def site_likelihood(tree, site, Q, verbose=False):
    charmap = {"A":0, "G":1, "C":2, "T":3} 
    
    # recursion function
    def walk(n):
        # if we are at a leaf, the recursion ends (we know the state)
        if n.is_leaf():
            state = charmap[site[n.name]]  # get the state
            pvec = np.zeros(4)             # initialize the probability vector
            pvec[state] = 1                # set p[state] = 1
            if verbose:
                print("{:>10}".format(n.name), pvec)
            return pvec
        # we are at an internal node
        else:
            node_pvec = np.ones(4)         # initialize a probability vector of ones
            for c in n.children:           # recurse over the child nodes 
                tmp_pvec = np.zeros(4)  
                child_pvec = walk(c)       # recursion! get the p-vector at the child node
                P = linalg.expm(Q*c.dist)  # compute the P matrix
                for i in range(4):
                    tmp_pvec += P[i, :] * child_pvec[i]
                node_pvec *= tmp_pvec
            if verbose: 
                print("{:>10}".format(",".join(n.get_leaf_names())), node_pvec)
            return node_pvec
        
    # do recursion and return the probability vector at the root
    return sum(walk(tree))

Now we had the likelihood for a given site (i.e. a column of an alignment), but what about a full alignment? Since we assume independent evolution across the sites of the alignment, the alignment likelihood is just the product of the site-wise likelihoods (or the sum when working in log space).

def aln_likelihood(tree, aln, Q):
    lhood = 0
    for i in range(len(list(aln.values())[0])):
        site = {k: v[i] for k, v in aln.items()}
        site_lhood = np.log(site_likelihood(tree, site, Q))
        lhood += site_lhood
    return lhood

The maximum likelihood tree¶

So this gives us the likelihood of a data set given that we know everything, i.e. the topology, branch lengths and substitution model parameters. That is of course silly, because we are looking for eactly these things. Under the ML principle we can however use this likelihood to try and find the parameters, branch lengths and topology that maximize this likelihood. That is, we try to solve the following problem:

$$\hat{T}, \hat{b}, \hat{\theta} = \mathrm{argmax}_{T,b,\theta}\mathcal{L}(T, b, \theta|D, \mathcal{M})$$

Heuristic tree search¶

Tree topologies however provide a big challenge for the optimization. Finding the topology that maximizes the likelihood is an intractable combinatorial problem. Recall that tree space is huge, so computing the likelihood for all possible tree topologies is certainly no option.

Instead, we have to resort to a heuristic search through tree space. The basic approach is a greedy hill climbing strategy. This works as follows (where $\theta$ denotes all parameters of the model (excluding the topology), i.e. branch lengths and parameters of the substitution models):

Tree topology $T$, branch lengths $b$ and parameters of substitution model $\theta$

  1. Start with a random tree $T$, find the ML estimates for $\theta$ and $b$ and compute the likelihood $L$
  2. Modify the tree using some kind of tree rearrangement move to get a new topology $T'$
  3. Search for the ML estimates of $\theta$ and $b$ given $T'$ and compute the likelihood $L'$ of the new tree $T'$
  4. If $L' > L$ set $T \leftarrow T' $ and $L \leftarrow L'$
  5. If some convergence condition is met, stop, else go back to step 2

Model selection¶

Nested models - the LRT¶

In all the above, we have assumed that the substitution model $\mathcal{M}$ was given. There are however a tremendous number of substitution models that you could use! As already suggested in our comparison of JC69 with JC69+$\Gamma$, we need a way to find the model that best fits our data set. Intuitively, one could think of it as follows: we are searching for the substitution model that, when we simulate random sequences from the substitution process along the true tree topology with the true branch lengths starting from the true ancestral sequence, will give sequences at the tips that look a lot like our data set.

A general technique to perform model selection in a maximum likelihood setting is simply to compare the likelihood values for different candidate models using a so-called likelihood ratio test (LRT).

Consider two models, for example the Jukes & Cantor model (JC69) and the more complex Kimura model (K80). We can compute the maximum likelihood estimate and maximal likelihood values, call these maximal likelihood values $l_{JC69}$ and $l_{K80}$ repectively. We are interested whether the K80 model provide a better fit to the sequence data or whether the simpler JC69 model will do the job. The ratio of the maximal likelihood values

$$ \Lambda = \frac{l_{K80}}{l_{JC69}} $$

tells us how many times more likely the K80 model is to have generated the data compared to the JC69 model.

Note that simply knowing how many times more likely one model is to be the hypothetical true model does not tell us whether this difference is statistically significant. The likelihood ratio can be converted in a test statistic, the LRT, which allows us to assess the statistical significance of the difference in maximal likelihood values.

By Wilks' theorem, we know that the test statistic

$$ D = -2\mathrm{log}\Bigg( \frac{l_{1}}{l_{0}}\Bigg) \sim \chi^2_{k_1 - k_0}$$

where

  • $l_1$ is the likelihood under model 1 with $k_1$ parameters
  • $l_0$ is the likelihood under model 0 with $k_0$ parameters
  • $k_1 > k_0$

The LRT only works however for nested models, i.e. the simpler model is a special case of the more parameter-rich model. Furthermore, Wilks' theorem only holds when the parameters are not at the boundary of the parameter space.

Information criteria¶

Some of the models are not nested, in that case we can use information criteria such as the Akaike information criterion (AIC) or Bayesian information criterion (BIC).

$$ AIC = 2k - 2ln(l) $$

The model with the lowest AIC value provides a better fit to the data. The value

$$ \mathrm{exp}\Big(\frac{AIC_{min} - AIC_i}{2}\Big) $$

is proportional to the probability that the $i^{th}$ model provides an equally well fit as the model with minimal AIC value.

ModelFinder¶

IQ-TREE by default uses ModelFinder to automatically select the best fitting model based on AIC or BIC in an efficient way. ModelFinder works as follows (with default settings):

  1. Find a reasonable tree $T$ (inferred using parsimony)
  2. Find the maximal likelihood $L(D|T,\mathcal{M}_i)$ and associated parameters of the substitution model $\mathcal{M}_i$ for all $i$ where $\{\mathcal{M}\}_{1,\cdots,i}$ is a list of substitution models (and models of rate heterogeneity, e.g. $\Gamma$)
  3. Identify $\mathcal{M}_\mathrm{optimal}$ using the $BIC$ criterion
